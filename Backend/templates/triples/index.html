About Us


As our Bacholers dissertation, we decided on coducting our research about the interesting topic of the implementation of knowledge graphs for machine learning in cybersecurity.
Get to know more about us and our work through our Mission, our Vision, and our Achievements!

MISSION

The main objective of this end of bachelors studies project, is to obtain a knowledge graph extracted from several popular datasets. This is because, the large and diverse amount of data, hides very useful knowledge relations in cybersecurity. The strategy adopted is based on RDF graphs, and NoSQL stores .

VISION

Nowadays, Data Mining (DM), Machine Learning (ML), and Artificial Intelligence (AI) tools are widely applied in cybersecurity, aiming to help produce powerful, and intelligent intrusion detection systems.

Our vision is to develop a tool, that provides an effective visual representation for intelligent vulnerability detection, which will support existing Machine Learning solutions. This will be done through the use of Knowledge Graphs.

ACHIEVEMENTS

We were successfully able to:

- Develop a cybersecurity knowledge graph obtained from existing datasets.

- Allow visualisation of different Cybersecurity Knowledge Graphs by admin and non-admin users.

- Allow modifications to be performed on the Knowledge Graphs by admin users

- Allow the extraction of information from the Knowledge Graphs by admin users, using a set of queries.

NSL-KDD Dataset

The NSL-KDD dataset consists of network traffic data captured from a simulated environment, including various types of intrusive activities.

This dataset graph representation contains several features extracted from network packets, such as connection attributes, protocol-related information, and host-based features.

DNS
Dataset

The DNS over HTTPS (DoH) dataset contains the collection of data that captures various aspects of DoH usage.

This dataset graph representation includes information such as the source IP addresses of DNS queries, the corresponding domain names being queried, timestamps of the queries, the DoH resolver used, and other relevant metadata.

To be able to perform INSERT , MODIFY , and DELETE operations use:


curl -X (method) "(our current site ip adresse)/api/put/(QUERY)/(graph-name)/(subject)/(predicat)/(object)/" -H "Authorization: Token (your token key)"


NOTE!

Make sure that the method matches with the Query


use:
